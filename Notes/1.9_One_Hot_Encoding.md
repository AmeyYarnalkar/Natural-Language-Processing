
# **One Hot Encoding (NLP Notes)**

## **Core Idea**

Convert each unique word into a vector where only one index is **1** and all others are **0**.
Each word gets a **unique position** in the vector space.

---

## **Problem**

Machines cannot understand text directly.
We need to convert words into **numerical form**, but:

* Words have no inherent numeric meaning
* We need a **simple and deterministic mapping**

---

## **How It Works**

1. Build a **vocabulary** of all unique words
   Example:

   ```
   ["I", "love", "NLP"]
   ```

2. Assign each word an index:

   ```
   I → 0
   love → 1
   NLP → 2
   ```

3. Create vectors of length = vocab size

| Word | Vector    |
| ---- | --------- |
| I    | [1, 0, 0] |
| love | [0, 1, 0] |
| NLP  | [0, 0, 1] |

---

## **Examples**

Sentence:

```
"I love NLP"
```

Encoded:

```
I     → [1, 0, 0]
love  → [0, 1, 0]
NLP   → [0, 0, 1]
```

Another example:

```
"I love AI"
```

New vocabulary:

```
["I", "love", "NLP", "AI"]
```

Vectors change size:

```
AI → [0, 0, 0, 1]
```

---

## **Code Snippet (Python)**

```python
from sklearn.preprocessing import OneHotEncoder

sentences = [["I"], ["love"], ["NLP"]]

encoder = OneHotEncoder(sparse_output=False)
encoded = encoder.fit_transform(sentences)

print(encoded)
```

Manual approach:

```python
vocab = ["I", "love", "NLP"]

one_hot = {}

for i, word in enumerate(vocab):
    vector = [0] * len(vocab)
    vector[i] = 1
    one_hot[word] = vector

print(one_hot)
```

---

## **Variations**

* Character-level one-hot encoding
* Word-level one-hot encoding
* Sentence-level (bag of words style)

---

## **Limitations**

This is where things start breaking:

1. **No semantic meaning**

   * "king" and "queen" are completely unrelated

2. **High dimensional**

   * Large vocab → huge vectors

3. **Sparse vectors**

   * Mostly zeros → inefficient

4. **No context awareness**

   * Same word = same vector everywhere

---


## **When to Use / Not Use**

### Use when:

* Learning basics of NLP
* Very small datasets
* Simple models

### Avoid when:

* Real-world NLP systems
* Large vocabulary
* Need semantic understanding

---

## **Pipeline Placement**

```
Text → Tokenization → One Hot Encoding → Model
```

---

## **Use Cases**

* Basic classification problems
* Teaching/learning NLP fundamentals
* Simple ML pipelines

---

## **Key Insight**

One Hot Encoding is a **starting point**, not a solution.
It treats words as **independent symbols**, not meaningful entities.

👉 That’s why modern NLP moved to embeddings like **Word2Vec, GloVe, BERT**
