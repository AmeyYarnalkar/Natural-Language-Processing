# NLP Notes — POS Tagging


## 1. Core Idea

**Part-of-Speech (POS) Tagging** is the process of assigning a grammatical label (e.g., noun, verb, adjective) to each word in a sentence based on its role and context.

---

## 2. Problem

Words can have multiple meanings depending on usage:

* “running” can be a verb or a noun
* “better” can be an adjective or adverb

Challenges:

* ambiguity in word meaning
* incorrect interpretation without context
* downstream tasks (like lemmatization) may fail without role information

---

## 3. How it Works

```text id="q4k8m1"
Input Text
 → Tokenization
 → Context-based tagging model
 → Assign POS tag to each token
 → Output (word, tag) pairs
```

Process:

1. Tokenize the sentence
2. Analyze each word with surrounding context
3. Use a trained model or rules to assign the most likely tag

---

## 4. Examples

Sentence:
"She is running fast"

POS Tags:

* She → PRP (Pronoun)
* is → VBZ (Verb)
* running → VBG (Verb - gerund)
* fast → RB (Adverb)

---

Another example:

"Running is fun"

* Running → NN (Noun)
* is → VBZ
* fun → JJ

---

## 5. Code Snippet

Using NLTK:

```python id="1z7m5k"
from nltk.tokenize import word_tokenize
from nltk import pos_tag

# nltk.download('punkt')
# nltk.download('averaged_perceptron_tagger')

text = "She is running faster than anyone"

tokens = word_tokenize(text)
tags = pos_tag(tokens)

print(tags)
```

---

## 6. Variations

* **Rule-based Tagging**
  Uses predefined grammar rules

* **Statistical Tagging**
  Uses probabilities based on training data

* **Neural Tagging**
  Uses deep learning models for higher accuracy

---

## 7. Limitations

* Ambiguity in complex sentences
* Errors in unseen or rare contexts
* Language-dependent behavior
* Not always 100% accurate

---

## 8. Comparison

POS Tagging vs Tokenization:

* Tokenization:

  * Splits text into units
  * Does not assign meaning

* POS Tagging:

  * Assigns grammatical role
  * Adds contextual understanding

---

## 9. When to Use / When Not to Use

Use when:

* performing lemmatization
* analyzing sentence structure
* building advanced NLP systems

Avoid when:

* working with simple keyword-based tasks
* context is not important

---

## 10. Pipeline Placement

```text id="6y8h2r"
Raw Text
 → Tokenization
 → POS Tagging
 → Lemmatization / Parsing
 → Feature Extraction
 → Model
```

---

## 11. Use Cases

* Text parsing
* Information extraction
* Chatbots
* Machine translation
* Grammar checking systems

---

## 12. Key Insight

POS tagging adds structure to text by identifying the role of each word. This role-based understanding is essential for accurate language processing and improves the effectiveness of downstream NLP tasks.
