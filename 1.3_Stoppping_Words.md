# NLP Notes — Stopwords

## 1. Core Idea

**Stopwords** are commonly occurring words in a language that carry minimal semantic value and are often removed during preprocessing to reduce noise.

---

## 2. Problem

Text data contains many high-frequency words (e.g., *the, is, and*) that:

* do not help differentiate documents
* increase data size and redundancy
* dilute the importance of meaningful words

Keeping them can:

* reduce model efficiency
* weaken signal-to-noise ratio

---

## 3. How it Works

```text
Input Text
 → Tokenization
 → Compare tokens with stopword list
 → Remove matching words
 → Output filtered tokens
```

Process:

1. Tokenize the text
2. Load a predefined or custom stopword list
3. Filter out tokens present in the list

---

## 4. Examples

Sentence:
"The cat is sitting on the mat"

After stopword removal:
["cat", "sitting", "mat"]

---

Counter-example (important):

Sentence:
"This movie is not good"

After stopword removal:
["movie", "good"]

Meaning is altered because "not" was removed.

---

## 5. Code Snippet

Using NLTK:

```python
from nltk.corpus import stopwords
from nltk.tokenize import word_tokenize

# nltk.download('stopwords')
# nltk.download('punkt')

text = "The cat is sitting on the mat"

stop_words = set(stopwords.words('english'))
tokens = word_tokenize(text)

filtered = [word for word in tokens if word.lower() not in stop_words]

print(filtered)
```

---

## 6. Variations

* **Standard Stopwords**
  Predefined lists (e.g., NLTK English stopwords)

* **Custom Stopwords**
  Domain-specific words added manually

* **Dynamic Stopwords**
  Words removed based on frequency thresholds in a dataset

---

## 7. Limitations

* Can remove important context words (e.g., *not*)
* Not suitable for all NLP tasks
* Language and domain dependent
* May oversimplify text representation

---

## 8. Comparison

Stopword Removal vs Keeping Stopwords:

* Removal:

  * Reduces noise
  * Improves efficiency
  * May lose context

* Keeping:

  * Preserves full meaning
  * Better for contextual tasks
  * Increases data size

---

## 9. When to Use / When Not to Use

Use when:

* text classification
* information retrieval
* topic modeling

Avoid when:

* sentiment analysis
* question answering
* language modeling

---

## 10. Pipeline Placement

```text
Raw Text
 → Tokenization
 → Stopword Removal
 → Lemmatization / Stemming
 → Feature Extraction
 → Model
```

---

## 11. Use Cases

* Search engines
* Document clustering
* Keyword extraction
* Basic NLP pipelines

---

## 12. Key Insight

Stopwords are not inherently useless. They are task-dependent. Removing them improves efficiency, but careless removal can distort meaning and reduce model performance.
