# Optimizers – Detailed Notes

## Core Idea

Optimizers are responsible for **updating weights and bias using gradients** computed from backpropagation.

### General Update Rule

wₜ = wₜ₋₁ − η · (dL/dw)

Where:

* η → learning rate
* dL/dw → gradient

---

# Basic Terminologies

* **Epoch** → One full pass over the entire dataset
* **Iteration** → One update step (using one batch)
* **Batch Size** → Number of samples used in one iteration

---

# Types of Optimizers

---

## 1. Batch Gradient Descent (Gradient Descent)

### Idea

* Compute loss using **entire dataset**
* Update weights once per epoch

### Characteristics

* Stable and smooth updates
* Requires **high memory**
* Slow for large datasets

### Summary

* Updates after full dataset
* Works in **epochs**

---

## 2. Stochastic Gradient Descent (SGD)

### Idea

* Update weights using **one data point at a time**

### Characteristics

* Fast updates
* High variance (noisy updates)
* Can escape local minima

### Drawback

* Convergence is **unstable and oscillatory**

---

## 3. Mini-Batch Gradient Descent

### Idea

* Uses **small batches of data** (e.g., 32, 64)

### Characteristics

* Balanced approach
* Faster than batch GD
* More stable than SGD

### Drawback

* Some noise still exists

### Summary

> Most commonly used in practice

---

## 4. SGD with Momentum

### Problem it Solves

* SGD updates are noisy and oscillate

### Idea

* Maintain **memory of past gradients**

### Concept

* Use **Exponential Weighted Average (EWA)**

### Formulas

Velocity update:
vₜ = β vₜ₋₁ + (1 − β) (dL/dw)

Weight update:
wₜ = wₜ₋₁ − η · vₜ

### Where

* β → momentum factor (usually ~0.9)

### Benefits

* Reduces oscillations
* Faster convergence

---

## 5. Adagrad (Adaptive Gradient Descent)

### Problem it Solves

* Fixed learning rate is not optimal

### Idea

* Use **different learning rates for each parameter**

### Formula

Accumulated gradient:
Gₜ = Σ (dL/dw)²

Update:
wₜ = wₜ₋₁ − (η / √(Gₜ + ε)) · (dL/dw)

### Benefits

* Good for sparse data

### Drawback

* Learning rate keeps decreasing
* Can become too small → stops learning

---

## 6. RMSProp

### Problem it Solves

* Adagrad’s learning rate decays too much

### Idea

* Use **exponential moving average instead of full sum**

### Formula

Sₜ = β Sₜ₋₁ + (1 − β)(dL/dw)²

Update:
wₜ = wₜ₋₁ − (η / √(Sₜ + ε)) · (dL/dw)

### Benefits

* Stable learning rate
* Works well in practice

---

## 7. Adam (Adaptive Moment Estimation)

### Idea

* Combines:

  * Momentum (first moment)
  * RMSProp (second moment)

### Formulas

First moment (momentum):
mₜ = β₁ mₜ₋₁ + (1 − β₁)(dL/dw)

Second moment:
vₜ = β₂ vₜ₋₁ + (1 − β₂)(dL/dw)²

Update:
wₜ = wₜ₋₁ − (η / √(vₜ + ε)) · mₜ

### Benefits

* Fast convergence
* Adaptive learning rates
* Works well for most problems

👉 Default optimizer in practice

---

# Key Insights

* Batch GD → stable but slow
* SGD → fast but noisy
* Mini-batch → best balance
* Momentum → smooth updates
* Adagrad → adaptive but decays too much
* RMSProp → fixes Adagrad
* Adam → combines best of both

---

# Final Practical Rule

* Start with → **Adam**
* Try → **SGD + Momentum** if needed

---

# One Line Summary

Optimizers differ in how they use gradients to update weights efficiently, balancing speed, stability, and convergence.
