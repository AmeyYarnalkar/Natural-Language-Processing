
# Error, Loss, and Weight Update 

## Predicted vs Actual

* The model produces a **predicted value / class**
* We also have the **actual class** (true value)

---

## Error Calculation

* We compare predicted output with actual output
* This difference is called **error**

To measure this properly, we use:

* **Loss function** → for a single data point
* **Cost function** → for a batch of data points

(No need for formulas right now)

---

## Goal

The main objective is:

> **Reduce the error as much as possible**

---

## Key Insight

The only way to reduce error is:

> Change how input features influence the output

And this influence is controlled by:

* **weights** (and bias)

So:

> Better weights → better predictions → lower error

---

## What Needs to Be Updated

* We need to update:

  * weights
  * bias

So that:

* important features get more influence
* less useful features get reduced influence

---

## Role of Optimizer

The unit responsible for updating weights is called:

> **Optimizer**

Its job is:

* take the error
* decide how weights should change
* update them in a way that reduces error

---

## Important Clarification

You said:

> “reduce error by any weight possible”

Refined version:

> We don’t randomly change weights
> We update them in a **directed way** so that error decreases

That direction is what makes learning effective.

---

## Big Question (Next Step)

Now the important part is:

* How do we know:

  * which weight to change?
  * how much to change it?
  * in which direction to change it?

## One Line Summary

We measure how wrong the model is using loss, and then an optimizer updates weights in a directed way to reduce that error over time.
