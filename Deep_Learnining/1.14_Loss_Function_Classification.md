# Loss Functions for Classification

## 1. Binary Cross Entropy (Log Loss)

| Key              | Value                                                                     |
| ---------------- | ------------------------------------------------------------------------- |
| Definition       | Measures difference between predicted probability and actual binary label |
| Formula (Single) | −[y log(ŷ) + (1 − y) log(1 − ŷ)]                                          |
| Formula (Batch)  | −(1/n) Σ [yᵢ log(ŷᵢ) + (1 − yᵢ) log(1 − ŷᵢ)]                              |
| Behavior         | Penalizes wrong confident predictions heavily                             |
| Advantages       | Works well with sigmoid; Strong gradient for wrong predictions            |
| Disadvantages    | Sensitive to extreme probabilities (log(0) issue)                         |

---

## 2. Categorical Cross Entropy

| Key              | Value                                                                           |
| ---------------- | ------------------------------------------------------------------------------- |
| Definition       | Measures difference between predicted probability distribution and actual class |
| Formula (Single) | − Σ yᵢ log(ŷᵢ)                                                                  |
| Formula (Batch)  | −(1/n) Σ Σ yᵢⱼ log(ŷᵢⱼ)                                                         |
| Behavior         | Focuses on probability of correct class                                         |
| Advantages       | Works with softmax; Standard for multi-class classification                     |
| Disadvantages    | Requires one-hot encoding; Sensitive to incorrect confident predictions         |

---

## 3. Sparse Categorical Cross Entropy

| Key              | Value                                                                     |
| ---------------- | ------------------------------------------------------------------------- |
| Definition       | Same as categorical cross entropy but uses class index instead of one-hot |
| Formula (Single) | − log(ŷ_correct_class)                                                    |
| Formula (Batch)  | −(1/n) Σ log(ŷ_correct_class)                                             |
| Behavior         | Only considers probability of correct class                               |
| Advantages       | Memory efficient; Simpler labels                                          |
| Disadvantages    | Less intuitive than one-hot; Same sensitivity as cross entropy            |

---

## 4. Hinge Loss

| Key              | Value                                               |
| ---------------- | --------------------------------------------------- |
| Definition       | Used for margin-based classification (like SVM)     |
| Formula (Single) | max(0, 1 − y·ŷ)                                     |
| Formula (Batch)  | (1/n) Σ max(0, 1 − yᵢ·ŷᵢ)                           |
| Behavior         | Penalizes predictions inside margin                 |
| Advantages       | Encourages margin separation; Robust for some tasks |
| Disadvantages    | Not probabilistic; Less common in deep learning     |

---

## Intuition (Very Important)

* **Cross Entropy** → checks how good your probability prediction is
* **Binary Cross Entropy** → for 2 classes
* **Categorical Cross Entropy** → for multiple classes
* **Sparse Cross Entropy** → same but simpler labels
* **Hinge Loss** → focuses on margin, not probability

---

## When to Use What

### Binary Classification

* **Sigmoid + Binary Cross Entropy**

---

### Multi-Class Classification

* **Softmax + Categorical Cross Entropy**

OR

* **Softmax + Sparse Categorical Cross Entropy** (when labels are integers)

---

### Special Cases

* Use **Hinge Loss** for margin-based models (rare in DL)

---

## Final Practical Rule

> Binary → BCE
> Multi-class → CCE / Sparse CCE

---

## One Line Summary

Classification losses measure how well predicted probabilities match actual classes, with cross-entropy being the standard choice.
