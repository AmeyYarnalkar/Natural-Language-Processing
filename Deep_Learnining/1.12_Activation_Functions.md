
# Sigmoid

| Key                  | Value                                                            |
| -------------------- | ---------------------------------------------------------------- |
| Purpose / Definition | Used for binary classification; converts values into probability |
| Formula              | σ(x) = 1 / (1 + e⁻ˣ)                                             |
| Range                | (0, 1)                                                           |
| Derivative           | σ'(x) = σ(x)(1 − σ(x))                                           |
| Derivative Range     | (0, 0.25)                                                        |
| Advantages           | Smooth function; Output interpretable as probability             |
| Disadvantages        | Vanishing gradient; Not zero-centered                            |

---

# Tanh

| Key                  | Value                                                 |
| -------------------- | ----------------------------------------------------- |
| Purpose / Definition | Improved version of sigmoid; zero-centered activation |
| Formula              | tanh(x) = (eˣ − e⁻ˣ) / (eˣ + e⁻ˣ)                     |
| Range                | (−1, 1)                                               |
| Derivative           | 1 − tanh²(x)                                          |
| Derivative Range     | (0, 1)                                                |
| Advantages           | Zero-centered output; Stronger gradients than sigmoid |
| Disadvantages        | Vanishing gradient; Saturation at extremes            |

---

# ReLU

| Key                  | Value                                                          |
| -------------------- | -------------------------------------------------------------- |
| Purpose / Definition | Default activation for hidden layers; introduces non-linearity |
| Formula              | f(x) = max(0, x)                                               |
| Range                | [0, ∞)                                                         |
| Derivative           | 1 (x > 0), 0 (x ≤ 0)                                           |
| Derivative Range     | {0, 1}                                                         |
| Advantages           | Fast computation; Avoids vanishing gradient (positive region)  |
| Disadvantages        | Dying ReLU problem; Zero gradient for negative values          |

---

# Leaky ReLU

| Key                  | Value                                                     |
| -------------------- | --------------------------------------------------------- |
| Purpose / Definition | Fixes dying ReLU by allowing small negative slope         |
| Formula              | f(x) = x (x > 0), αx (x ≤ 0)                              |
| Range                | (−∞, ∞)                                                   |
| Derivative           | 1 (x > 0), α (x ≤ 0)                                      |
| Derivative Range     | {α, 1}                                                    |
| Advantages           | Prevents dead neurons; Allows gradient in negative region |
| Disadvantages        | α is manually chosen; May not always be optimal           |

---

# Parametric ReLU (PReLU)

| Key                  | Value                                              |
| -------------------- | -------------------------------------------------- |
| Purpose / Definition | Learns the negative slope instead of fixing it     |
| Formula              | f(x) = x (x > 0), αx (x ≤ 0)                       |
| Range                | (−∞, ∞)                                            |
| Derivative           | 1 (x > 0), α (x ≤ 0)                               |
| Derivative Range     | (learned α, 1)                                     |
| Advantages           | Adaptive slope; Better flexibility than Leaky ReLU |
| Disadvantages        | Extra parameter to learn; Risk of overfitting      |

---

# ELU

| Key                  | Value                                                                   |
| -------------------- | ----------------------------------------------------------------------- |
| Purpose / Definition | Smooth ReLU variant with negative outputs                               |
| Formula              | f(x) = x (x > 0), α(eˣ − 1) (x ≤ 0)                                     |
| Range                | (−α, ∞)                                                                 |
| Derivative           | 1 (x > 0), f(x) + α (x ≤ 0)                                             |
| Derivative Range     | (0, 1]                                                                  |
| Advantages           | Reduces vanishing gradient; Produces negative values (better centering) |
| Disadvantages        | Computationally expensive; More complex than ReLU                       |

---

# Softmax

| Key                  | Value                                                                         |
| -------------------- | ----------------------------------------------------------------------------- |
| Purpose / Definition | Converts outputs into probability distribution for multi-class classification |
| Formula              | softmax(xᵢ) = eˣⁱ / Σ eˣʲ                                                     |
| Range                | (0, 1), sum = 1                                                               |
| Derivative           | Depends on all outputs (coupled derivatives)                                  |
| Derivative Range     | Not fixed                                                                     |
| Advantages           | Produces probability distribution; Works well for multi-class problems        |
| Disadvantages        | Computationally expensive; Sensitive to large values                          |

---

## Final Note (Keep This in Mind)

* Sigmoid / Tanh → conceptually important but limited
* ReLU family → practical default for hidden layers
* Softmax → output layer for multi-class

---



## Activation Function – Decision Guide

### 1. Step Function

**When to use:**

* Mostly **not used in modern neural networks**

**Why:**

* Not differentiable
* Cannot be used with gradient-based learning

👉 Only useful for:

* Basic perceptron (theoretical understanding)

---

### 2. Sigmoid

**When to use:**

* **Output layer for binary classification**

**Why:**

* Gives probability between 0 and 1

**Avoid using in:**

* Hidden layers (causes vanishing gradient)

---

### 3. Tanh

**When to use:**

* Sometimes in hidden layers (older practice)

**Why:**

* Zero-centered (better than sigmoid)

**But:**

* Still suffers from vanishing gradient

👉 So not preferred in deep networks today

---

### 4. ReLU

**When to use:**

* **Default choice for hidden layers**

**Why:**

* Simple and fast
* Avoids vanishing gradient (for positive region)

---

### 5. Leaky ReLU

**When to use:**

* When you face **dying ReLU problem**

**Why:**

* Keeps small gradient for negative values

👉 Safer alternative to ReLU

---

### 6. PReLU / ELU

**When to use:**

* When you want better performance tuning

**Why:**

* More flexible / smoother variants

👉 Used in more advanced setups

---

### 7. Softmax

**When to use:**

* **Output layer for multi-class classification**

**Why:**

* Converts outputs into probabilities
* Sum = 1 → easy class selection

---

## Final Practical Rule (Very Important)

### Binary Classification

* Output layer → **Sigmoid**
* Hidden layers → **ReLU / Leaky ReLU**

---

### Multi-Class Classification

* Output layer → **Softmax**
* Hidden layers → **ReLU / Leaky ReLU**

---

### Hidden Layers (General Rule)

* Use **ReLU** by default
* Use **Leaky ReLU** if ReLU causes issues
