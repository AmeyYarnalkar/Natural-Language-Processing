# NLP Notes : Named Entity Recognition and Relationship Extraction


## 1. Core Idea

**Named Entity Recognition (NER)** identifies and classifies important entities (such as people, organizations, and locations) in text, while **Relationship Extraction** identifies how these entities are connected.

---

## 2. Problem

Raw text contains information in an unstructured form:

* Entities are not explicitly labeled
* Relationships between entities are implicit
* Machines cannot directly interpret “who did what”

Without structuring:

* information cannot be queried or analyzed effectively
* knowledge extraction becomes difficult

---

## 3. How it Works

```text id="9k3v2p"
Input Text
 → Tokenization
 → POS Tagging
 → Named Entity Recognition (identify entities)
 → Relationship Extraction (identify connections)
 → Structured Output
```

Process:

1. Detect entities (e.g., person, organization)
2. Classify them into predefined categories
3. Analyze sentence structure
4. Identify relationships using rules or models

---

## 4. Examples

Sentence:
"Elon Musk founded SpaceX in United States"

NER:

* Elon Musk → PERSON
* SpaceX → ORGANIZATION
* United States → LOCATION

Relationship:

* founded(Elon Musk → SpaceX)

---

Another example:

"Sachin Tendulkar played for India national cricket team"

Relationship:

* played_for(Sachin Tendulkar → India team)

---

## 5. Code Snippet

Using NLTK:

```python id="8n2p7x"
import nltk
from nltk.tokenize import word_tokenize
from nltk import pos_tag, ne_chunk

text = "Elon Musk founded SpaceX"

tokens = word_tokenize(text)
pos_tags = pos_tag(tokens)

tree = ne_chunk(pos_tags)

entities = []

for subtree in tree:
    if hasattr(subtree, 'label'):
        name = " ".join([word for word, tag in subtree.leaves()])
        label = subtree.label()
        entities.append((name, label))

print(entities)
```

---

## 6. Variations

* **Rule-based NER**
  Uses patterns and dictionaries

* **Statistical NER**
  Uses probabilistic models

* **Neural NER**
  Uses deep learning models (modern approach)

* **Relationship Extraction Methods**:

  * Rule-based (verb patterns)
  * Dependency parsing
  * Supervised learning models

---

## 7. Limitations

* Difficulty in detecting ambiguous entities
* Limited accuracy in complex sentences
* Relationship extraction is harder than entity detection
* Domain-specific entities may not be recognized

---

## 8. Comparison

NER vs POS Tagging:

* POS Tagging:

  * Assigns grammatical roles
  * Focuses on syntax

* NER:

  * Identifies real-world entities
  * Focuses on semantic meaning

---

## 9. When to Use / When Not to Use

Use when:

* extracting structured information from text
* building knowledge graphs
* analyzing documents or news

Avoid when:

* task does not require entity-level understanding
* simple keyword matching is sufficient

## 11. Use Cases

* Resume parsing
* News analysis
* Knowledge graph construction
* Question answering systems
* Chatbots

---

## 12. Key Insight

NER identifies “what entities exist,” while relationship extraction explains “how those entities are connected.” Together, they convert unstructured text into structured, machine-understandable knowledge.
