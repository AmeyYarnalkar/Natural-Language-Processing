# NLP Notes — Stemming

## 1. Core Idea

**Stemming** is the process of reducing words to a base form (stem) by removing suffixes or prefixes, without guaranteeing that the result is a valid dictionary word.

---

## 2. Problem

In text data, the same concept appears in multiple word forms:

* play, playing, played, plays
* study, studies, studying

Challenges:

* increases vocabulary size
* creates data sparsity
* prevents models from linking similar meanings

---

## 3. How it Works

```text id="6n3m5l"
Input Text
 → Tokenization
 → Apply stemming rules (remove suffix/prefix)
 → Output stemmed tokens
```

Process:

1. Tokenize the text
2. Apply rule-based transformations
3. Remove common suffixes (e.g., *ing, ed, ly*)
4. Return shortened word forms

---

## 4. Examples

| Original  | Stemmed |
| --------- | ------- |
| playing   | play    |
| studies   | studi   |
| running   | run     |
| happiness | happi   |

Observation:

* Output may not be valid words (e.g., *studi, happi*)

---

## 5. Code Snippet

Using NLTK:

```python id="u8r2xa"
from nltk.stem import PorterStemmer
from nltk.tokenize import word_tokenize

text = "running runs ran easily fairness"

ps = PorterStemmer()
tokens = word_tokenize(text)

stemmed = [ps.stem(word) for word in tokens]

print(stemmed)
```

---

## 6. Variations

* **Porter Stemmer**
  Balanced, widely used

* **Lancaster Stemmer**
  More aggressive, may over-stem

* **Snowball Stemmer**
  Improved version of Porter, supports multiple languages

---

## 7. Limitations

* Produces non-dictionary words
* Does not consider context
* Can over-stem or under-stem
* May distort meaning

---

## 8. Comparison

Stemming vs Lemmatization:

* Stemming:

  * Rule-based
  * Fast
  * Less accurate
  * Output may be invalid

* Lemmatization:

  * Dictionary-based
  * Slower
  * More accurate
  * Output is valid word

---

## 9. When to Use / When Not to Use

Use when:

* speed is important
* approximate normalization is sufficient
* working with large datasets

Avoid when:

* meaning accuracy is critical
* interpretability is required

---

## 10. Pipeline Placement

```text id="8whp0t"
Raw Text
 → Tokenization
 → Stopword Removal
 → Stemming
 → Feature Extraction
 → Model
```

---

## 11. Use Cases

* Search engines
* Text classification
* Information retrieval
* Basic NLP pipelines

---

## 12. Key Insight

Stemming is a fast normalization technique that reduces word variations, but it sacrifices linguistic correctness. It is useful when efficiency matters more than precision.
