# NLP Notes : Tokens and Tokenization


## 1. Core Idea

**Tokenization** is the process of breaking text into smaller units called **tokens**, which are the basic units a model can process.

---

## 2. Problem

Raw text is unstructured and cannot be directly used by machine learning models.

Challenges:

* Sentences are continuous strings
* Punctuation and spacing vary
* Models require discrete units

Without tokenization:

* No consistent input format
* No way to map text to numbers

---

## 3. How it Works

```text
Input Text
 → Split based on rules (spaces, punctuation, patterns)
 → Produce tokens
```

Process:

1. Take raw sentence
2. Apply splitting logic (simple or advanced)
3. Output list of tokens

---

## 4. Examples

Sentence:
"I love NLP!"

* Word-level tokens:
  ["I", "love", "NLP", "!"]

* Subword tokens:
  ["I", "love", "NL", "P"]

* Character tokens:
  ["I", "l", "o", "v", "e"]

---

## 5. Code Snippet

Using NLTK:

```python
from nltk.tokenize import word_tokenize, sent_tokenize

text = "I love NLP. It is interesting."

# Sentence tokenization
sentences = sent_tokenize(text)

# Word tokenization
words = word_tokenize(text)

print(sentences)
print(words)
```

---

## 6. Variations

* **Word Tokenization**
  Splits text into words

* **Sentence Tokenization**
  Splits text into sentences

* **Subword Tokenization**
  Breaks words into smaller parts (used in modern models)

* **Character Tokenization**
  Splits into individual characters

---

## 7. Limitations

* Ambiguity in splitting (e.g., "can't")
* Language-specific challenges
* Loss of meaning in naive splitting
* Subword tokenization may produce non-intuitive tokens

---

## 8. Comparison

Word Tokenization vs Subword Tokenization:

* Word Tokenization:

  * Simple
  * Large vocabulary
  * Struggles with unknown words

* Subword Tokenization:

  * Handles unseen words
  * Smaller vocabulary
  * More complex

---

## 9. When to Use / When Not to Use

Use tokenization:

* in all NLP pipelines
* before any text processing or modeling

Choice depends on task:

* Word-level → traditional ML
* Subword-level → deep learning models

Not applicable:

* outside text processing

---

## 10. Pipeline Placement

```text
Raw Text
 → Tokenization
 → Cleaning (Stopwords, Lemmatization)
 → Feature Extraction
 → Model
```

---

## 11. Use Cases

* Text preprocessing
* Search engines
* Chatbots
* Machine translation
* Sentiment analysis

---

## 12. Key Insight

Tokenization is not just splitting text. It defines how language is broken into units, and that choice directly impacts how effectively a model understands and processes text.
